# Установка ExchangeKit на VPS

Автоматическая установка одним скриптом: Node.js 20, PostgreSQL 16, nginx,
systemd-сервис бэкенда и опционально HTTPS (Let's Encrypt).

Скрипт [install.sh](install.sh) разворачивает всё нативно (без Docker):

- бэкенд (`server/`, Express + tsx) — как systemd-сервис `exchangekit-api` на порту `4000`;
- фронтенд собирается в `dist/` и раздаётся через nginx;
- nginx проксирует `/api` → `127.0.0.1:4000`, отдаёт SPA, при желании включает HTTPS;
- таблицы БД и админ-аккаунт создаются самим сервером при первом запуске.

---

## 1. Требования

| Что | Значение |
|-----|----------|
| ОС | Ubuntu 24.04 LTS (чистый сервер) |
| RAM | от 1 ГБ (рекомендуется 2 ГБ) |
| Диск | от 10 ГБ |
| Доступ | root (или пользователь с `sudo`) |
| Домен | A-запись домена должна указывать на IP сервера |

Доменное имя обязательно — установка спросит его в самом начале. Для HTTPS домен
уже должен резолвиться на этот сервер (иначе Let's Encrypt не выдаст сертификат).

---

## 2. Подключение к серверу

```bash
ssh root@IP_ВАШЕГО_СЕРВЕРА
```

---

## 3. Получение кода проекта

Репозиторий приватный. Выберите один из вариантов.

**Вариант А — клонировать через SSH** (нужен доступ к репозиторию с сервера):

```bash
apt-get update && apt-get install -y git
git clone git@github.com:pachugonis/exchange.git
cd exchange
```

**Вариант Б — скопировать код со своей машины** (если на сервере нет доступа к GitHub):

```bash
# выполняется на ЛОКАЛЬНОЙ машине, из каталога проекта
rsync -a --exclude node_modules --exclude dist ./ root@IP_СЕРВЕРА:/root/exchange/
```

> Каталог `.git` нужно сохранить — он используется кнопкой «Обновить» в админке
> (обновление делается через `git pull`).

## 4. Запуск установки

Из корня проекта, от root:

```bash
sudo bash INSTALL/install.sh
```

Скрипт идемпотентен — его можно безопасно запускать повторно: пароль БД и
`JWT_SECRET` сохраняются между запусками, пользователи не разлогиниваются.

### Что спросит мастер установки

1. **Домен сайта** — например `exchange.example.com`.
2. **E-mail администратора** — логин для входа в админ-панель.
3. **Пароль администратора** — минимум 8 символов (вводится дважды).
4. **HTTPS (Let's Encrypt)** — `y`, если домен уже указывает на сервер; иначе `N`
   (можно включить позже).
5. **SMTP** — данные почтового сервера для писем (верификация e-mail, сброс пароля).
   Если оставить хост пустым — письма будут писаться в лог сервера вместо отправки.
6. **Etherscan API key** — для авто-отслеживания оплаты в сети Ethereum (ETH/ERC20).
   BTC/LTC/TRON отслеживаются без ключа; без ключа ETH подтверждается вручную.

После сводки конфигурации подтвердите запуск нажатием `Enter`.

### Шаг с deploy-ключом (для обновлений)

На шаге 5 скрипт сгенерирует SSH deploy-ключ и покажет **публичную** часть.
Чтобы работала кнопка «Обновить» в админке (она тянет код через `git pull`):

1. Скопируйте показанный публичный ключ.
2. В GitHub: **репозиторий → Settings → Deploy keys → Add deploy key**
   (достаточно прав read-only).
3. Вернитесь в терминал и нажмите `Enter` — скрипт проверит доступ.

Если обновления через админку не нужны — нажмите `s`, чтобы пропустить проверку.

## 5. Что происходит во время установки (10 шагов)

1. Сбор конфигурации.
2. Системные пакеты (nginx, postgresql, curl, ufw, openssl…).
3. Node.js 20 LTS.
4. Системный пользователь `exchangekit` и копирование файлов в `/opt/exchangekit`.
5. Deploy-ключ для обновлений из приватного репозитория.
6. PostgreSQL: роль `exchange_user` и база `exchange`.
7. Генерация `/opt/exchangekit/.env` (секреты, DATABASE_URL, SMTP, seed-админ).
8. `npm install` + сборка фронтенда (`npm run build` → `dist/`).
9. systemd-сервис `exchangekit-api` + правило sudoers для кнопки «Обновить».
10. Конфиг nginx, UFW (SSH/HTTP/HTTPS), при необходимости — сертификат Let's Encrypt.

Лог установки сохраняется в `INSTALL/installation.log`.

## 6. После установки

В конце скрипт покажет адреса и учётные данные. Они также сохраняются в
`/opt/exchangekit/.credentials` (после переноса данных в надёжное место файл
рекомендуется удалить).

- Сайт: `https://ВАШ_ДОМЕН`
- Админ-панель: `https://ВАШ_ДОМЕН/admin/login`

### Полезные команды

```bash
# Логи API в реальном времени
journalctl -u exchangekit-api -f

# Рестарт / статус бэкенда
systemctl restart exchangekit-api
systemctl status exchangekit-api

# Логи nginx
tail -f /var/log/nginx/error.log

# Обновление приложения вручную
bash /opt/exchangekit/INSTALL/update.sh
```

---

## 7. Включить HTTPS позже

Если на установке HTTPS пропустили, после того как домен заработает на сервере:

```bash
certbot --nginx -d ВАШ_ДОМЕН
```

---

## 8. Диагностика

| Симптом | Что проверить |
|---------|---------------|
| API не отвечает на `/health` | `journalctl -u exchangekit-api -n 50` — частая причина: недоступна БД или ошибка в `.env` |
| Сайт не открывается | `nginx -t`, затем `systemctl status nginx`; проверьте, что UFW пропускает 80/443 |
| Не выдаётся сертификат | домен должен указывать на сервер; повторите `certbot --nginx -d ВАШ_ДОМЕН` |
| Кнопка «Обновить» не работает | deploy-ключ не добавлен в GitHub либо `origin` не в SSH-форме |
| Письма не приходят | при пустом `SMTP_HOST` письма идут в лог; задайте SMTP в `/opt/exchangekit/.env` и `systemctl restart exchangekit-api` |

Файл окружения для ручной правки: `/opt/exchangekit/.env`
(после изменений — `systemctl restart exchangekit-api`).
