## Методы обновления

ExchangeKit поддерживает два метода обновления:

### 1. Обновление из пакета (рекомендуется)
- Самый безопасный метод с автоматическим резервным копированием и откатом
- Работает офлайн
- Включает все необходимые файлы для обновления

### 2. Обновление из Git (для продвинутых пользователей)
- Требуется доступ к Git репозиторию
- Загружает последние изменения из удалённого репозитория
- Требуется подключение к интернету

## Подготовка обновления

### Для обновления из пакета

**На вашей машине разработки:**

1. Внесите изменения в код
2. Тщательно протестируйте
3. Создайте пакет обновления:

```bash
cd INSTALL
bash create-update-package.sh v1.2.0
```

Это создаст пакет в `dist/updates/exchangekit-update-v1.2.0.tar.gz`

4. Передайте пакет на ваш сервер:

```bash
scp dist/updates/exchangekit-update-v1.2.0.tar.gz user@ваш-сервер:/tmp/
```

### Для обновления из Git

**На вашем сервере:**

1. Убедитесь, что Git репозиторий настроен в `/opt/exchangekit`
2. Убедитесь, что у вас есть доступ для pull из репозитория

## Применение обновления

### Метод 1: Обновление из пакета

```bash
# Подключитесь к вашему серверу по SSH
ssh user@ваш-сервер

# Запустите скрипт обновления с путём к пакету
sudo bash /opt/exchangekit/scripts/update.sh /tmp/exchangekit-update-v1.2.0.tar.gz
```

### Метод 2: Обновление из Git

```bash
# Подключитесь к вашему серверу по SSH
ssh user@ваш-сервер

# Запустите скрипт обновления без параметров
sudo bash /opt/exchangekit/scripts/update.sh
```

## Что происходит во время обновления

Скрипт обновления выполняет следующие шаги:

1. **Проверка** - Проверяет текущую установку
2. **Резервное копирование** - Создаёт резервную копию конфигурации и данных
3. **Остановка сервисов** - Останавливает все Docker контейнеры
4. **Обновление файлов** - Заменяет файлы приложения
5. **Пересборка** - Пересобирает Docker контейнеры
6. **Миграция БД** - Выполняет миграции базы данных
7. **Запуск сервисов** - Запускает обновлённые контейнеры
8. **Проверка** - Проверяет успешность обновления
9. **Очистка** - Удаляет старые резервные копии (сохраняет последние 5)

### Автоматическое резервное копирование

Перед каждым обновлением скрипт создаёт резервную копию, содержащую:
- Конфигурацию `.env`
- Файлы конфигурации администратора
- База данных лицензий
- Конфигурация Docker compose
- Конфигурация Nginx

Резервные копии хранятся в `/opt/exchangekit-backups/backup_ГГГГММДД_ЧЧММСС/`

## Откат

Если обновление не удалось, скрипт автоматически откатывается к предыдущей версии.

### Ручной откат

Если вам нужно вручную откатиться после успешного обновления:

```bash
# Список доступных резервных копий
ls -lt /opt/exchangekit-backups/

# Выберите резервную копию и восстановите её
sudo bash /opt/exchangekit/scripts/rollback.sh /opt/exchangekit-backups/backup_20231215_143022
```

```bash

Enter your choice (1-2):
```

Или задайте язык через переменную окружения:

```bash
# Для английского языка
export LANG_CODE=en
sudo bash /opt/exchangekit/scripts/update.sh /tmp/update-package.tar.gz
```

## Отслеживание версий

Каждый пакет обновления включает файл VERSION с:
- Номером версии
- Временем сборки
- Git commit hash (если доступен)

Проверка текущей версии:
```bash
cat /opt/exchangekit/VERSION
```

## Устранение неполадок

### Обновление не удалось с "Установка не найдена"
**Решение:** Убедитесь, что ExchangeKit установлен в `/opt/exchangekit`

### Сервисы не запускаются после обновления
**Решение:** 
```bash
# Проверьте логи
docker compose logs -f

# При необходимости, откатитесь
ls -lt /opt/exchangekit-backups/
# Используйте последнюю резервную копию
```

### Ошибки миграции базы данных
**Решение:**
```bash
# Проверьте статус базы данных
docker exec exchangekit-postgres psql -U exchange_user -d exchange_db -c "\dt"

# Ручная миграция
sudo bash /opt/exchangekit/scripts/migrate-database.sh
```

### Ошибки прав доступа
**Решение:**
```bash
# Установите правильные права
sudo chown -R root:root /opt/exchangekit
sudo chmod +x /opt/exchangekit/scripts/*.sh
```

### Сборка Docker не удалась
**Решение:**
```bash
# Очистите кэш Docker
docker system prune -af

# Повторите обновление
sudo bash /opt/exchangekit/scripts/update.sh <путь-к-пакету>
```

## Лучшие практики

1. **Всегда делайте резервную копию вручную перед крупными обновлениями:**
   ```bash
   sudo tar -czf /root/exchangekit-manual-backup.tar.gz /opt/exchangekit
   ```

2. **Сначала тестируйте обновления в тестовой среде**

3. **Планируйте обновления на периоды низкой нагрузки**

4. **Отслеживайте логи после обновления:**
   ```bash
   docker compose logs -f
   ```

5. **Храните хотя бы одну проверенную резервную копию вне сервера**

6. **Документируйте пользовательские изменения** перед обновлением

## Регулярные обновления

Для регулярных обновлений (исправления безопасности, мелкие исправления):

```bash
# Быстрое обновление (автоматизировано)
sudo bash /opt/exchangekit/scripts/update.sh /tmp/update-package.tar.gz

# Проверка
docker compose ps
curl -I http://localhost
```

## Поддержка

Если у вас возникли проблемы во время обновления:
1. Проверьте логи ошибок
2. Просмотрите резервную копию в `/opt/exchangekit-backups/`
3. Обратитесь в поддержку с деталями ошибки
4. Сохраняйте резервные копии до подтверждения стабильности обновления

---

**Примечание:** Всегда убедитесь, что у вас есть свежая резервная копия перед выполнением обновлений!
