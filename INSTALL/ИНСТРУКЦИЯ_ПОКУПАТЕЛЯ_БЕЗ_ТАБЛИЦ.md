# Установка ExchangeKit — инструкция для покупателя

Спасибо за покупку! Эта инструкция проведёт вас через установку платформы на ваш
сервер от начала до конца. Технических навыков почти не требуется — нужно лишь
аккуратно выполнить шаги и ответить на вопросы установщика.

Вся установка занимает 10–20 минут.

---

## Что вы получили после оплаты

1. Файл `install.sh` — установщик.
2. Лицензионный ключ вида `LIC-XXXX-XXXX-XXXX-XXXX`.
3. E-mail, на который оформлена лицензия.

Исходный код скачивать и клонировать не нужно — установщик сам загрузит его с
нашего сервера в виде защищённого подписанного архива.

---

## Что нужно подготовить

**VPS-сервер.** Чистая Ubuntu 24.04 LTS. Минимум 1 ГБ оперативной памяти (лучше
2 ГБ) и 10 ГБ диска. Подойдёт любой хостинг: Timeweb, Beget, Hetzner,
DigitalOcean и подобные.

**Доступ root.** Логин и пароль (или SSH-ключ) для входа на сервер под
пользователем `root`.

**Домен.** Например `exchange.example.com`. У домена должна быть A-запись,
указывающая на IP-адрес вашего сервера.

**Интернет на сервере.** Сервер должен иметь доступ в интернет — он скачивает
код платформы с лицензионного сервера.

### Как направить домен на сервер

В панели управления доменом (там, где вы его купили) создайте A-запись:

- Имя/Host: ваш домен или поддомен (например `exchange` или `@`).
- Значение/IP: IP-адрес вашего VPS.

Изменения DNS применяются от нескольких минут до нескольких часов. Это нужно
сделать до установки, если хотите сразу включить HTTPS.

---

## Шаг 1. Подключитесь к серверу

С вашего компьютера откройте терминал и подключитесь по SSH:

```bash
ssh root@IP_ВАШЕГО_СЕРВЕРА
```

Введите пароль сервера. Вы окажетесь в командной строке сервера.

На Windows можно использовать программу PuTTY или встроенный терминал
(PowerShell) — команда та же.

---

## Шаг 2. Загрузите установщик на сервер

Вариант А — скопировать файл с компьютера. Выполняется в терминале вашего
компьютера, не на сервере:

```bash
scp install.sh root@IP_ВАШЕГО_СЕРВЕРА:/root/
```

Вариант Б — создать файл прямо на сервере. Откройте редактор:

```bash
nano /root/install.sh
```

Вставьте содержимое присланного файла, нажмите `Ctrl+O` (сохранить), `Enter`,
затем `Ctrl+X` (выход).

---

## Шаг 3. Сделайте скрипт исполняемым

После загрузки файла на сервер дайте ему право на запуск:

```bash
chmod +x /root/install.sh
```

Команда `chmod +x` помечает файл как программу, которую можно запускать. Это
нужно сделать один раз. Проверить можно командой `ls -l /root/install.sh` — в
правах должны появиться буквы `x` (например `-rwxr-xr-x`).

---

## Шаг 4. Запустите установку

Запустите скрипт от имени root:

```bash
sudo /root/install.sh
```

Либо, если не делали файл исполняемым на предыдущем шаге, можно запустить так
(этот способ работает всегда, без `chmod`):

```bash
sudo bash /root/install.sh
```

Запустится мастер установки. Он задаст несколько вопросов.

---

## Шаг 5. Ответьте на вопросы мастера

**Домен сайта.** Ваш домен, например `exchange.example.com`. К нему привяжется
лицензия.

**Лицензионный ключ.** Ключ `LIC-...`, который вы получили.

**E-mail лицензии.** E-mail, на который оформлена лицензия.

**E-mail администратора.** Ваша почта для входа в админ-панель.

**Пароль администратора.** Придумайте надёжный пароль (минимум 8 символов).
Обязательно запомните его.

**Настроить HTTPS сейчас?** Введите `y`, если домен уже указывает на сервер
(рекомендуется). Если DNS ещё не настроен — `N`, включите HTTPS позже.

**SMTP.** Настройки почтового сервера для писем (подтверждение e-mail, сброс
пароля). Если оставить пустым — письма будут писаться в лог, а не отправляться.
Можно настроить позже.

**Etherscan API key.** Ключ для автоматического отслеживания оплат в сети
Ethereum (ETH/ERC20). Необязательно — можно пропустить нажатием Enter. BTC, LTC
и TRON отслеживаются и без него.

**AMLBot access ID и key.** Ключи для AML-проверки отправителей транзакций.
Необязательно — можно пропустить. Ключи выдаются в личном кабинете AMLBot.

В конце мастер покажет сводку. Проверьте данные и нажмите `Enter` для запуска.

Поля SMTP, Etherscan и AMLBot можно безопасно пропустить сейчас и добавить
позже — установка от этого не пострадает.

---

## Шаг 6. Дождитесь завершения

Установщик автоматически сделает всё сам (примерно 5–10 минут):

- установит необходимые программы (веб-сервер, базу данных и другие);
- активирует вашу лицензию и привяжет её к домену;
- скачает и проверит подлинность кода платформы;
- настроит сайт, базу данных и (при выборе `y`) HTTPS-сертификат.

В конце вы увидите зелёное сообщение «Установка завершена» с адресами и данными
для входа.

---

## Готово! Что дальше

После установки доступны:

- Сайт: `https://ВАШ_ДОМЕН`
- Админ-панель: `https://ВАШ_ДОМЕН/admin/login`
- Вход: e-mail и пароль администратора, которые вы указали.

Эти же данные сохранены на сервере в файле `/opt/exchangekit/.credentials`.
Сохраните их в надёжном месте и удалите этот файл:

```bash
rm /opt/exchangekit/.credentials
```

Войдите в админ-панель, чтобы настроить курсы валют, направления обмена,
реквизиты и оформление сайта.

## Полезно знать

### Если не включили HTTPS при установке

Когда домен начнёт указывать на сервер, включите HTTPS командой:

```bash
certbot --nginx -d ВАШ_ДОМЕН
```

### Обновления

Когда выйдет новая версия, обновитесь одним из способов:

- Из админ-панели — кнопка «Обновить» (самый простой способ).
- Вручную на сервере:

```bash
bash /opt/exchangekit/INSTALL/update.sh
```

Обновление безопасно: загружается только проверенный подписанный код, данные
сайта при этом сохраняются.

### Перезапуск и логи (на случай проблем)

```bash
systemctl restart exchangekit-api      # перезапустить сайт
systemctl status exchangekit-api       # проверить статус
journalctl -u exchangekit-api -f       # смотреть логи в реальном времени
```

## Частые вопросы и проблемы

**«Не удалось активировать лицензию».** Проверьте, что ключ и e-mail введены без
опечаток, а домен совпадает с тем, на который оформлена лицензия. Сервер должен
иметь доступ в интернет.

**Сайт не открывается по домену.** Убедитесь, что A-запись домена указывает на
IP сервера и DNS успел обновиться. Проверьте состояние командой
`systemctl status exchangekit-api`.

**Письма не приходят.** Если при установке вы не указали SMTP — письма не
отправляются. Настройте SMTP позже (см. ниже) и перезапустите сайт.

**Забыли пароль администратора.** Данные сохранены в файле
`/opt/exchangekit/.credentials`, если вы его не удалили.

**HTTPS не работает.** Включите его командой `certbot --nginx -d ВАШ_ДОМЕН`
после настройки DNS.

### Как изменить настройки после установки

Все настройки (SMTP, ключи API и прочее) хранятся в файле
`/opt/exchangekit/.env`. Откройте его, отредактируйте нужные строки и
перезапустите сайт:

```bash
nano /opt/exchangekit/.env
systemctl restart exchangekit-api
```

## Нужна помощь?

Если что-то пошло не так — напишите в поддержку и приложите:

- ваш домен и лицензионный ключ;
- последние строки лога: вывод команды `journalctl -u exchangekit-api -n 50`;
- файл лога установки `INSTALL/installation.log` рядом со скриптом.

Мы поможем разобраться.
