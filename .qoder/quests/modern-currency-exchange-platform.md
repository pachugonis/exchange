# Современная Платформа Обмена Валют

## Обзор Проекта

Современный веб-сайт обмена валют с интуитивным интерфейсом, темной темой и поддержкой множества направлений обмена (криптовалюты, электронные кошельки, банковские карты, наличные). Платформа предоставляет пользователям возможность быстро и безопасно обменивать различные виды валют с прозрачным отображением курсов и комиссий.

## Цели и Задачи

### Бизнес-Цели
- Предоставить удобный и безопасный сервис обмена валют
- Обеспечить прозрачность процесса обмена для пользователей
- Минимизировать время выполнения транзакций
- Повысить доверие клиентов через отображение сертификатов и политик безопасности

### Технические Цели
- Создать высокопроизводительное одностраничное приложение (SPA)
- Обеспечить отзывчивый дизайн для всех устройств
- Реализовать интуитивный пользовательский интерфейс
- Обеспечить масштабируемость архитектуры

## Технологический Стек

### Frontend Framework
- **React 18+** с **TypeScript** для типобезопасной разработки
- **Vite** как инструмент сборки для быстрой разработки и оптимизированной production-сборки

### Стилизация
- **Tailwind CSS** для utility-first подхода к стилизации
- Кастомная конфигурация темной темы с поддержкой переключения

### Управление Состоянием
- **Zustand** для глобального состояния приложения (курсы валют, активные заявки, настройки темы)
- **React Context API** для локального состояния (форма обмена, пошаговая навигация)

### Маршрутизация
- **React Router v6** для клиентской навигации

### UI Компоненты
- **Lucide React** для иконок
- Кастомные компоненты интерфейса, построенные на Tailwind CSS

### Дополнительные Библиотеки
- Библиотека для валидации форм
- Библиотека для отображения уведомлений (toast notifications)
- Библиотека для работы с графиками (история курсов)
- Библиотека для форматирования чисел и валют

### Компонентная Архитектура

#### Принципы Организации
- **Атомарный дизайн**: UI компоненты организованы от простых к сложным
- **Разделение ответственности**: бизнес-логика отделена от представления
- **Переиспользуемость**: общие компоненты в `/ui`, специфичные в тематических папках
- **Композиция**: сложные компоненты собираются из простых

#### Иерархия Компонентов

**Уровень 1: UI Компоненты** (базовые элементы интерфейса)
- Button, Input, Select, Card, Badge, Modal, Toast

**Уровень 2: Функциональные Компоненты** (специфичная логика)
- ExchangeForm, CurrencySelector, RateDisplay

**Уровень 3: Композитные Компоненты** (секции страниц)
- HeroSection, FeaturesSection, TestimonialsSection

**Уровень 4: Страницы** (полные представления)
- Home, Exchange, About, Rules

### 2. Модуль Формы Обмена

#### Структура Формы

**Раздел "Отдаю"**:
- Выпадающий список валют с поиском
- Иконка валюты
- Поле ввода суммы
- Индикатор минимальной/максимальной суммы

**Раздел "Получаю"**:
- Выпадающий список валют с поиском
- Иконка валюты
- Автоматически рассчитанная сумма (read-only)
- Индикатор резерва

**Информационный Блок**:
- Текущий курс обмена
- Размер комиссии
- Итоговая сумма к получению
- Время резервирования курса (таймер 15 минут)

**Действия**:
- Кнопка "Обменять" (переход к процессу обмена)
- Иконка обмена направлений (swap)

#### Currency Selector
**Функциональность**:
- Поиск валюты по названию/коду
- Группировка валют по типам:
  - Криптовалюты
  - Электронные кошельки
  - Банковские карты
  - Наличные
- Отображение иконки, названия и кода валюты
- Индикация доступности (резерв > 0)

#### Расчет Курса
**Логика**:
- Получение актуального курса из store
- Применение комиссии сервиса
- Учет минимальных и максимальных сумм
- Проверка наличия резерва
- Автоматический пересчет при изменении суммы или валюты
- Debounce при вводе для оптимизации

### 3. Модуль Процесса Обмена (Пошаговая Форма)

#### Общая Структура
- Индикатор прогресса (5 шагов)
- Навигация между шагами
- Кнопки "Назад" и "Далее"
- Валидация перед переходом к следующему шагу
- Сохранение данных в LocalStorage

#### Шаг 1: Проверка Заявки
**Цель**: Подтверждение параметров обмена

**Отображение**:
- Валюта и сумма "отдаю"
- Валюта и сумма "получаю"
- Курс обмена
- Комиссия
- Таймер резервирования курса
- Кнопка "Подтвердить и продолжить"

**Действия**:
- Проверка актуальности курса
- Проверка резерва
- Возможность вернуться к калькулятору

#### Шаг 2: Контактные Данные
**Цель**: Сбор контактной информации для связи

**Поля**:
- Email (обязательное)
- Telegram username (опционально)
- Промокод (опционально)

**Валидация**:
- Формат email
- Формат Telegram username (начинается с @)

#### Шаг 3: Реквизиты Кошельков
**Цель**: Получение платежных данных

**Блок "Откуда отправить"**:
- Реквизиты отправителя (зависят от типа валюты)
- Для крипты: адрес кошелька, сеть
- Для карты: номер карты, имя держателя
- Для электронного кошелька: номер счета

**Блок "Куда получить"**:
- Реквизиты получателя
- Аналогичная структура в зависимости от типа валюты

**Валидация**:
- Формат адреса криптокошелька
- Формат номера карты (Luhn algorithm)
- Формат номера электронного кошелька
- Соответствие выбранной валюте

#### Шаг 4: Подтверждение Заявки
**Цель**: Финальная проверка всех данных

**Отображение**:
- Все введенные данные в структурированном виде:
  - Параметры обмена
  - Контактные данные
  - Реквизиты
- Чекбокс согласия с правилами обмена
- Чекбокс согласия с AML/KYC политикой
- Генерация уникального ID заявки

**Действия**:
- Валидация всех данных
- Кнопка "Создать заявку"
- Отправка данных (mock API)

#### Шаг 5: Статус Обмена
**Цель**: Информирование о ходе выполнения заявки

**Элементы**:
- Уникальный ID заявки (с кнопкой копирования)
- Индикатор статуса (progress bar)
- Текущий этап:
  - Ожидание оплаты
  - Оплата получена
  - Проверка транзакции
  - Отправка средств
  - Завершено
- Реквизиты для оплаты (с кнопкой копирования)
- Таймер времени на оплату
- Инструкции по оплате
- Кнопка "Я оплатил"
- Кнопка "Связаться с поддержкой"

**Обновление Статуса**:
- Polling (проверка статуса каждые 10 секунд)
- Симуляция изменения статуса через mock API
- Уведомления при изменении статуса

### 4. Модуль Управления Валютами

#### Типы Валют

**Криптовалюты**:
- Bitcoin (BTC)
- Ethereum (ETH)
- Tether USDT (TRC20, ERC20, BEP20)
- USD Coin (USDC)
- Другие популярные криптовалюты

**Электронные Кошельки**:
- Payeer (USD, EUR, RUB)
- Perfect Money (USD, EUR)
- AdvCash (USD, EUR, RUB)
- Wise (USD, EUR, GBP)

**Банковские Карты**:
- Visa/Mastercard RUB
- Visa/Mastercard USD
- Visa/Mastercard EUR

**Наличные**:
- По городам (Москва, Санкт-Петербург, и т.д.)

#### Структура Данных Валюты

Каждая валюта содержит:
- Уникальный код
- Название (русское и английское)
- Тип (crypto, ewallet, card, cash)
- Иконка
- Минимальная сумма обмена
- Максимальная сумма обмена
- Текущий резерв
- Признак активности
- Доступные сети (для криптовалют)

#### Управление Курсами

**Источник Данных**:
- Mock data с имитацией реальных курсов
- Возможность подключения к реальным API в будущем

**Обновление Курсов**:
- Периодическое обновление (каждые 30 секунд)
- Имитация небольших колебаний курса
- Индикация направления изменения (вверх/вниз)

**Расчет Курса Обмена**:
- Базовый курс валютной пары
- Применение комиссии сервиса
- Учет резерва (может влиять на курс)

### 5. Модуль Управления Заявками

#### Создание Заявки

**Генерация ID**:
- Уникальный идентификатор заявки (например, EX-20240115-A1B2C3)
- Формат: префикс + дата + случайный код

**Сохранение Данных**:
- Параметры обмена
- Контактные данные
- Реквизиты
- Timestamp создания
- Начальный статус

**Хранение**:
- LocalStorage для возможности продолжить заявку
- Отправка на backend (mock API)

#### Отслеживание Заявки

**Страница Отслеживания**:
- Ввод ID заявки
- Отображение полной информации о заявке
- Текущий статус
- История изменений статуса
- Доступные действия

**Статусы Заявки**:
- Создана (waiting_payment)
- Ожидание подтверждения оплаты (payment_pending)
- Оплата получена (payment_received)
- Проверка транзакции (verification)
- Отправка средств (sending)
- Завершена (completed)
- Отменена (cancelled)
- Возврат средств (refund)

#### Уведомления

**Типы Уведомлений**:
- Заявка создана
- Оплата получена
- Средства отправлены
- Обмен завершен
- Требуется действие пользователя
- Ошибка или проблема

**Каналы Уведомлений**:
- Toast notifications внутри приложения
- Email (имитация отправки)
- Telegram bot (опционально, для будущей реализации)

## Управление Состоянием

### Zustand Stores

#### Exchange Store
**Ответственность**: Управление данными об обмене

**State**:
- Список всех валют
- Текущие курсы обмена
- Резервы валют
- Выбранная пара для обмена
- Рассчитанная сумма

**Actions**:
- Загрузка списка валют
- Обновление курсов
- Выбор валюты отдаю/получаю
- Расчет суммы обмена
- Проверка резерва

#### Order Store
**Ответственность**: Управление заявками

**State**:
- Текущая заявка (данные формы)
- История заявок пользователя
- Статус текущей заявки

**Actions**:
- Создание новой заявки
- Обновление данных заявки
- Изменение статуса заявки
- Загрузка заявки по ID
- Сохранение в LocalStorage

#### Theme Store
**Ответственность**: Управление темой интерфейса

## Валидация Данных

### Валидация Форм

#### Email
- Регулярное выражение для проверки формата
- Проверка на пустое значение
- Максимальная длина

#### Telegram Username
- Начинается с @
- Допустимые символы: буквы, цифры, подчеркивание
- Длина от 5 до 32 символов

#### Суммы
- Числовое значение
- Больше минимальной суммы
- Меньше максимальной суммы
- Меньше или равно резерву
- Формат с учетом десятичных знаков

#### Адрес Криптокошелька
- Проверка формата в зависимости от валюты
- BTC: длина, префикс, checksumma
- ETH: длина 42, начинается с 0x
- USDT: зависит от сети (TRC20, ERC20, BEP20)

#### Номер Банковской Карты
- Алгоритм Луна (Luhn algorithm)
- Длина 16-19 цифр
- Только цифры (с возможностью пробелов для отображения)

#### Номер Электронного Кошелька
- Формат зависит от платежной системы
- Payeer: P + цифры
- Perfect Money: U + цифры
- AdvCash: буквы и цифры с @

- Ошибки валидации полей

- Технические ошибки (ошибки парсинга)

## Глоссарий

**Обмен валют** - процесс конвертации одной валюты в другую

**Курс обмена** - соотношение стоимости двух валют

**Резерв** - доступное количество валюты для обмена

**Комиссия** - плата сервиса за проведение обмена

**AML (Anti-Money Laundering)** - политика противодействия отмыванию денег

**KYC (Know Your Customer)** - процедура проверки личности клиента

**Криптокошелек** - программа или сервис для хранения криптовалюты

**Электронный кошелек** - платежная система для онлайн-платежей

**Mock data** - имитационные данные для разработки и тестирования

**Polling** - периодическая проверка изменений через регулярные запросы

**Toast notification** - всплывающее уведомление

**LocalStorage** - механизм хранения данных в браузере пользователя

**Responsive design** - адаптивный дизайн для разных размеров экранов

**Code splitting** - разделение кода на меньшие части для оптимизации загрузки

**Lazy loading** - отложенная загрузка контента по мере необходимости

**SEO (Search Engine Optimization)** - оптимизация для поисковых систем
